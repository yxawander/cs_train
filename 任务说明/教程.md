## 1. Shell 简介

Shell 是一种提供用户与操作系统交互的命令行解释器。它接收用户输入的命令，并调用操作系统提供的功能来执行这些命令。

Shell 既可以作为交互式命令行工具，也可以作为脚本编程环境。它广泛用于 Unix 和 Linux 系统中，在其他操作系统中也有类似实现。

为了实现一个简易版本的自定义 Shell，需要掌握以下前置知识：

- 进程控制
- 进程等待
- 进程程序替换
- 环境变量读取
- 字符串切割
- 内建命令处理

整体实现思路如下：

1. 显示命令提示符。
2. 读取用户输入的命令字符串。
3. 将命令字符串切割成命令名和参数。
4. 判断是否为内建命令。
5. 如果是内建命令，则由 Shell 自身执行。
6. 如果不是内建命令，则创建子进程并执行外部程序。
7. 等待命令执行结束，继续下一轮输入。

## 2. 准备阶段

在正式实现 Shell 之前，需要先准备头文件、环境变量读取函数、宏定义和全局变量。

### 2.1 头文件

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <string.h>
#include <fcntl.h>
#include <stdbool.h>
#include <sys/wait.h>
#include <sys/types.h>
```

这些头文件会在后续功能中用到。

其中：

- `stdio.h`：用于输入输出，例如 `printf`、`fgets`。
- `stdlib.h`：用于环境变量、进程退出等功能，例如 `getenv`、`exit`。
- `unistd.h`：用于 Linux 系统调用，例如 `chdir`、`fork`、`execvp`。
- `string.h`：用于字符串处理，例如 `strlen`、`strcmp`、`strtok`。
- `fcntl.h`：用于文件控制相关操作。
- `stdbool.h`：用于布尔类型 `true` 和 `false`。
- `sys/wait.h`：用于进程等待，例如 `waitpid`。
- `sys/types.h`：用于系统数据类型，例如 `pid_t`。

### 2.2 获取环境变量的函数

命令提示符中通常会显示用户名、主机名和当前路径。  
这些信息可以从环境变量中读取。

```c
char* HOME()
{
    char* home = getenv("HOME");
    if(home == NULL) return "None";
    else return home;
}

const char* USER()
{
    char* user = getenv("USER");
    if(user == NULL) return "None";
    else return user;
}

const char* HOSTNAME()
{
    char* hostname = getenv("HOSTNAME");
    if(hostname == NULL) return "None";
    else return hostname;
}

const char* PWD()
{
    char* pwd = getenv("PWD");
    if(pwd == NULL) return "None";
    else return pwd;
}
```

这里使用 `const` 的原因是：返回的字符串只用于读取和显示，不需要修改。

函数含义如下：

- `HOME()`：获取用户主目录。
- `USER()`：获取当前用户名。
- `HOSTNAME()`：获取主机名。
- `PWD()`：获取当前工作目录。

### 2.3 宏定义和全局变量

```c
#define SIZE 1024
#define MAX_ARGC 64
#define SEP " "

char* argv[SIZE];
char env[SIZE];
char pwd[SIZE];
int lastcode = 0;
```

变量说明：

- `SIZE`：通用缓冲区大小。
- `MAX_ARGC`：参数个数上限。
- `SEP`：命令切割分隔符，这里使用空格。
- `argv`：保存命令切割后的参数。
- `env`：保存环境变量字符串。
- `pwd`：保存当前路径环境变量字符串。
- `lastcode`：保存上一条外部命令的退出码。

## 3. 实现 Shell

### 3.1 命令提示符

先观察 Linux 命令行中的提示符，例如：

```shell
ubuntu@VM-20-9-ubuntu:~/myShell$
```

从左到右分别表示：

- `ubuntu`：用户名。
- `VM-20-9-ubuntu`：主机名。
- `~/myShell`：当前路径。
- `$`：普通用户提示符。

这些信息可以通过环境变量查看：

```shell
echo $USER
echo $HOSTNAME
echo $PWD
```

示例输出：

```shell
ubuntu@VM-20-9-ubuntu:~/myShell$ echo $USER
ubuntu

ubuntu@VM-20-9-ubuntu:~/myShell$ echo $PWD
/home/ubuntu/myShell

ubuntu@VM-20-9-ubuntu:~/myShell$ echo $HOSTNAME
VM-20-9-ubuntu
```

在 C 语言中，可以通过 `getenv()` 函数获取环境变量。前面定义的 `USER()`、`HOSTNAME()` 和 `PWD()` 函数就是为了构造提示符。

```c
printf("[%s@%s%s]$", USER(), HOSTNAME(), PWD());
```

说明：

- `$` 表示普通用户。
- Linux 中 `root` 用户通常使用 `#` 作为提示符。

### 3.2 交互界面函数

Shell 的第一步是显示提示符，并读取用户输入。

```c
int Interactive(char in[], int size)
{
    printf("[%s@%s%s]$", USER(), HOSTNAME(), PWD());

    // 开始输入命令行
    fgets(in, size, stdin);

    // 去除最后的 '\n'
    in[strlen(in) - 1] = 0;

    return strlen(in);
}
```

为什么需要返回值？

返回值用于判断用户是否输入了内容。  
如果字符串长度为 `0`，说明用户直接按下回车，没有输入命令。

### 3.3 对命令字符串进行切割

获取用户输入后，需要把命令行字符串切割成命令名和参数。

例如用户输入：

```shell
ls -a -l
```

需要切割为：

```text
argv[0] = "ls"
argv[1] = "-a"
argv[2] = "-l"
argv[3] = NULL
```

这样后续才能把参数传给 `execvp()` 执行。

```c
void Split(char in[])
{
    int i = 0;

    argv[i++] = strtok(in, SEP);
    while(argv[i++] = strtok(NULL, SEP));

    if(strcmp(argv[0], "ls") == 0)
    {
        argv[i - 1] = (char*)"--color";
        argv[i] = NULL;
    }
}
```

这里使用了 `strtok()` 函数。

`strtok()` 的作用是按照指定分隔符切割字符串。  
本例中分隔符是空格，所以用户输入的命令会被拆分成多个参数。

#### 给 `ls` 增加颜色显示

代码中有一段特殊处理：

```c
if(strcmp(argv[0], "ls") == 0)
{
    argv[i - 1] = (char*)"--color";
    argv[i] = NULL;
}
```

这段代码的目的是给 `ls` 命令追加 `--color` 参数，使文件和目录能够用不同颜色显示。

### 3.4 处理内建命令

#### 什么是内建命令？

内建命令是由 Shell 自身直接提供和执行的命令，而不是系统上独立的可执行程序。

例如：

- `cd`
- `echo`
- `export`
- `pwd`
- `history`
- `exit`

这些命令通常涉及 Shell 自身状态，因此不能全部交给外部进程执行。

例如 `cd` 用于改变当前工作目录。  
如果在子进程中执行 `cd`，只会改变子进程的工作目录，父进程 Shell 的当前目录不会改变。  
因此 `cd` 必须由 Shell 自己实现。

### 3.5 内建命令实现

```c
int Bulidcmd()
{
    int ret = 0;

    // ret 为 0 表示非内建命令
    // ret 为 1 表示内建命令
    if(strcmp("cd", argv[0]) == 0)
    {
        ret = 1;

        char* target = argv[1];
        if(target == NULL) target = HOME();

        chdir(target);

        char temp[SIZE];
        getcwd(temp, SIZE);

        int result = snprintf(pwd, SIZE, "PWD=%s", temp);

        if(result < 0)
        {
            perror("snprintf()");
            exit(1);
        }
        else if(result >= SIZE)
        {
            fprintf(stderr, "error");
            exit(1);
        }

        putenv(pwd);
    }
    else if(strcmp("export", argv[0]) == 0)
    {
        ret = 0;

        if(argv[1])
        {
            strcpy(env, argv[1]);
            putenv(env);
        }
    }
    else if(strcmp("echo", argv[0]) == 0)
    {
        ret = 1;

        if(argv[1] == NULL)
        {
            printf("\n");
        }
        else
        {
            if(argv[1][0] == '$')
            {
                if(argv[1][1] == '?')
                {
                    printf("%d\n", lastcode);
                    lastcode = 0;
                }
                else
                {
                    char* e = getenv(argv[1] + 1);
                    if(e) printf("%s\n", e);
                }
            }
            else
            {
                printf("%s\n", argv[1]);
            }
        }
    }

    return ret;
}
```

> 注：原教程中函数名写作 `Bulidcmd`，更规范的拼写应为 `Buildcmd` 或 `BuildCmd`。为了和原代码保持一致，这里暂时不修改函数名。

### 3.6 常见内建命令说明

不同 Shell 提供的内建命令不完全相同。下面是一些常见内建命令：

- `cd`：更改当前工作目录。
- `exit`：退出当前 Shell。
- `echo`：打印文本到终端。
- `alias`：为命令创建别名。
- `set`：设置或显示 Shell 变量。
- `export`：将 Shell 变量导出为环境变量。
- `pwd`：显示当前工作目录。
- `history`：显示命令历史记录。
- `read`：从标准输入读取输入。
- `kill`：向进程发送信号，例如终止信号。
- `type`：显示命令类型，例如内建命令或外部命令。

本教程只实现了几个常见内建命令。

### 3.7 相关函数说明

#### chdir()

`chdir()` 是 C 语言中的系统调用函数，用于更改当前工作目录。

它的全称是 `change directory`。

常见用途：

- 实现 `cd` 命令。
- 改变进程当前工作路径。

#### getcwd()

`getcwd()` 用于获取当前工作目录的绝对路径。

它的全称是 `get current working directory`。

常见用途：

- 显示当前路径。
- 在 `cd` 后更新 `PWD` 环境变量。

#### putenv()

`putenv()` 用于设置或修改环境变量。

它的全称是 `put environment`。

常见用途：

- 修改 `PWD`。
- 实现 `export` 命令。

### 3.8 执行非内建命令

非内建命令需要通过子进程执行。

Linux 中常见流程是：

1. 使用 `fork()` 创建子进程。
2. 子进程使用 `execvp()` 进行程序替换。
3. 父进程使用 `waitpid()` 等待子进程结束。
4. 保存子进程退出码。

```c
void Execute()
{
    pid_t id = fork();

    if(id == 0)
    {
        // 让子进程执行外部命令
        execvp(argv[0], argv);
        exit(1);
    }

    int status = 0;
    pid_t rid = waitpid(id, &status, 0);

    if(rid == id)
    {
        lastcode = WEXITSTATUS(status);
    }
}
```

### 3.9 主函数

最后用一个循环把各个函数组织起来。

```c
int main()
{
    while(true)
    {
        // 1. 提供用户交互界面，并获取用户输入的命令字符串
        char commandline[SIZE];
        int n = Interactive(commandline, SIZE);

        if(n == 0) continue;

        // 2. 对字符串进行分割
        Split(commandline);

        // 3. 处理内建命令
        n = Bulidcmd();
        if(n) continue;

        // 4. 执行非内建命令
        Execute();
    }

    return 0;
}
```

整体执行流程：

1. 显示提示符。
2. 读取用户输入。
3. 判断用户是否输入内容。
4. 切割命令字符串。
5. 优先处理内建命令。
6. 如果不是内建命令，则执行外部程序。
7. 回到循环开始，继续等待下一条命令。

## 5. 当前缺点

当前自定义 Shell 还存在以下不足：

- 内建命令没有完整实现。
- 不支持删除等文件操作。
- 不支持管道。
- 不支持输入输出重定向。
- 不支持复杂引号解析。
- 不支持命令历史。
- 不支持通配符展开。
- 错误处理还比较简单。

## 7. 与 Windows 命令行解释器实训的对应关系

这篇教程实现的是 Linux 简易 Shell，因此和 Windows 实训题目存在 API 差异，但整体设计思想可以借鉴。

可以直接借鉴的部分：

- 命令循环结构。
- 命令提示符设计。
- 命令读取与字符串切割。
- 内建命令优先处理。
- 外部命令单独执行。
- 历史命令、退出命令等扩展思路。

Windows 实训中需要替换的部分：

- Linux 的 `fork()` 可替换为 Windows 的 `CreateProcess()`。
- Linux 的 `waitpid()` 可替换为 Windows 的 `WaitForSingleObject()`。
- Linux 的 `chdir()` 可替换为 Windows 的 `SetCurrentDirectory()`。
- Linux 的 `getcwd()` 可替换为 Windows 的 `GetCurrentDirectory()`。
- Linux 的进程信号机制可替换为 Windows 的 `OpenProcess()` 和 `TerminateProcess()`。
- Linux 的目录遍历函数可替换为 Windows 的 `FindFirstFile()` 和 `FindNextFile()`。

本实训题目要求实现的 `cd`、`dir`、`history`、`exit`、`tasklist`、`taskkill` 等命令，都可以按照“读取命令、解析参数、判断内建命令、调用系统 API、输出结果”的结构来完成。
